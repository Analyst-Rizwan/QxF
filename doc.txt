Executive Summary
The opportunity, the product, and the path forward
EduAI's School Tier is a focused extension of the core EduAI platform targeting Class 9–12 students under an NGO-led government education scheme. The goal: build genuine CS intuition early — not memorization for board exams, but the kind of foundational thinking that produces FAANG-level engineers over the next decade.
This document covers two things in one: (1) the scope and content flow of the NGO pilot module, and (2) how the school tier slots into EduAI's existing platform architecture. Both are designed to be built together, not sequentially.
Pilot Goal
Demonstrate full learning loop in one 45-minute module. Concept → Visualization → Sandbox → AI Tutor → Quiz.
Architecture Goal
Reuse EduAI's existing infra stack (Supabase, Render, Vercel). Add a School tier, School content spine, and coordinator dashboard.
01  |  The Opportunity
Why school-level CS infrastructure matters now
The Problem EduAI School Solves
Indian school CS education currently falls into two failure modes: rote theory for board exams (no building, no intuition) or expensive coaching that only metro, upper-income students can access. Neither produces engineers. Both produce students who arrive at college never having written a program that did something they actually wanted.
The NEP 2020 framework explicitly calls for computational thinking from Class 6 onward — but almost no infrastructure exists to deliver it at government school scale. That is the gap.
Why Now
The NGO-govt scheme pathway is the lowest-friction entry into this market. No tender process, no procurement cycle. A successful pilot converts directly into a funded rollout. Government schemes often have real per-student budget once a proof-of-concept is validated.
Why EduAI is Well-Positioned
Already building visualized CS content for college students — the tooling translates directly
Existing AI Tutor infrastructure (Haiku for chat, Sonnet for roadmaps) adapts to younger learners with prompt-level changes
Sandbox work (Judge0/Pyodide) already planned for EduAI college tier
Coimbatore base gives proximity to Tamil Nadu govt education network
Founder has direct domain familiarity as a recent CS graduate from the target student's future pathway
02  |  Pilot Module Scope
Introduction to Programming with Python — Class 9–10
Pilot Philosophy
One module. End-to-end. The NGO needs to see the full loop working — not a slide deck. Build the smallest complete experience that demonstrates concept → visual → hands-on → AI support → assessment. 30–45 minutes, completable in a single session.
Module: Introduction to Programming with Python
Target audience: Class 9–10, zero prior programming experience. Language: Python (no installation required, browser-based). Completion time: 30–45 minutes.
Learning Outcomes
Understand what a program is and why computers need instructions
Read and write simple Python statements (print, variables, input)
Understand what happens when code runs — step by step execution model
Modify a working program and observe the result
Ask the AI Tutor a question and get a useful answer
Module Flow — 5 Stages
1
Concept Hook  —  2–3 min
A 60-second animated explainer: what is a computer program? Use an everyday analogy — a recipe, or directions to a shop. No text walls. Ends with one question: ‘What do you think happens when you run this code?’
Animated SVG / Lottie  ·  Zero reading load
2
Visualized Execution  —  5–7 min
Show a 5-line Python program. Walk through it line by line — highlight the active line, show variable values changing in real time, show output appearing. Student controls: Step Forward, Step Back, Speed slider. No explanation text — the visual IS the explanation.
Interactive SVG step-through  ·  Student-paced
3
Guided Sandbox  —  10–15 min
Pre-loaded Python editor (Pyodide, browser-based). Three scaffolded micro-challenges. Challenge 1: Change a variable value, see output change. Challenge 2: Add your name to the print statement. Challenge 3: Write a program that asks your name and greets you. Each has a ‘Hint’ button before a ‘Solution’ button.
Pyodide sandbox  ·  Scaffolded  ·  No blank canvas
4
AI Tutor Checkpoint  —  5–10 min
After sandbox, surface 3 common questions students typically have at this point (‘Why do we need quotes?’, ‘What is a variable for?’, ‘Can Python do math?’). Student can tap one or type their own. AI responds in simple, age-appropriate language with a follow-up example. One free question included in pilot.
Claude Haiku  ·  Simplified system prompt  ·  Socratic tone
5
Quiz + Completion  —  5 min
5 MCQs. Concept-based, not syntax memorization. E.g., ‘What happens if you change this variable?’ with a code snippet. Score displayed. Completion badge shown. Progress written to NGO coordinator dashboard.
5 MCQs  ·  Concept-first  ·  Completion tracked
Content Tone and Language Principles
Core Rule
Never deficit-frame the student or their background. Frame every concept as access to something powerful, not remediation of a gap. Use Tamil Nadu everyday context where possible (auto, tiffin, recharge) as analogies, not just metro references.
All UI text in English, but analogy language can borrow from Tamil context
AI Tutor system prompt explicitly set to Class 9–10 reading level — no jargon unless introduced
Error messages in the sandbox must be human-readable, not raw Python tracebacks
Quiz feedback must explain why the answer is correct, not just mark right/wrong

04  |  Platform Architecture
How School Tier fits into EduAI
One Platform, Two Tiers
The core insight: EduAI's infrastructure (Supabase, Render, Vercel, AI routing) is content-agnostic. The School Tier is not a new platform — it is a new content spine + a new user role + a coordinator dashboard, running on the same stack.
Key Principle
Don't build twice. The AI Tutor, sandbox runner, visualizer framework, auth system, XP/streak engine, and payment infra are all shared. School Tier adds: School content DB, Student (School) role, Coordinator role, and simplified AI Tutor prompts.

New Components for School Tier
1. School Content Database (Supabase)
New tables alongside existing college content:
school_modules — title, grade level (9-10 / 11-12), order, status
school_lessons — concept text, visualizer config, sandbox starter code, quiz questions
school_progress — student_id, module_id, stage completed, score, timestamp
school_coordinators — NGO/school staff who can view batch-level dashboards
2. User Role: Student (School)
Extends the existing user roles (Student College, Admin). Key differences:
Onboarded via batch code (NGO issues a code, student enters it — no payment flow for pilot)
Sees School Tier content only, not College Tier content
AI Tutor quota: unlimited for pilot (NGO-funded), monthly limit post-pilot
Progress visible to their coordinator
3. Coordinator Dashboard
The NGO’s single most important feature. They need to demonstrate impact metrics to the govt scheme. Keep it minimal but complete:
Total students enrolled, active this week, modules completed
Per-student progress view: which module, which stage, quiz scores
Export to CSV for govt reporting
No coding required to use — web dashboard, login with email
4. School AI Tutor System Prompt
The AI Tutor uses Claude Haiku with a different system prompt for school students. Key prompt principles:
Respond at a Class 9-10 reading level (no technical jargon unless the concept requires it)
Use the Socratic method: ask ‘what do you think happens if...’ before giving answers
Always include a concrete, relatable example (not abstract)
Never say ‘that’s wrong’ — say ‘almost, let’s think about it differently’
If the student is stuck, break the problem into smaller steps

06  |  Commercial Model
How this becomes a sustainable business
The school tier has a fundamentally different commercial model from EduAI's B2C college subscription. The NGO pilot is free — it is a proof of concept. The commercial model activates post-pilot in three phases:
Phase 1: NGO Pilot
Free. 20–50 students. NGO covers any hard costs. Goal: validated impact data.
Phase 2: Scheme Rollout
Per-student fee paid by govt scheme budget. ₹75–100/student/month. 500–5,000 students.
Phase 3: Private School B2B
Sell to private schools and coaching institutes using pilot + scheme data as proof. ₹150–250/student/month.
Relationship to EduAI College
School students who complete Class 12 are natural EduAI college leads. The platform builds a pipeline, not just a product.
